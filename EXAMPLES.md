## 📊 判断示例

### 示例 1：应该关闭（评分 5）
```
最近消息：
- 客服：问题已经修复，您可以重新尝试
- 客户：好的，我试试
- 客服：有问题随时联系
- 客户：谢谢

分析：
+ 客户确认词"好的" (+3)
+ 客户感谢词"谢谢" (+2)
= 总分 5
→ strong_end_candidate → 关闭
```

### 示例 2：不应关闭（评分 -5）
```
最近消息：
- 客服：问题已经修复
- 客户：好的，还有个问题
- 客户：另外一个功能也有问题

分析：
+ 客户确认词"好的" (+3)
- 检测到新问题"还有个问题" (-5)
- 检测到新问题"另外一个功能" (-5)
= 总分 -7
→ not_end → 保留
```

### 示例 3：不确定（评分 2）
```
最近消息：
- 客服：我们会尽快处理
- 客户：嗯
- 客服：稍后答复您
- 客户：行

分析：
+ 我方承诺"稍后答复" (+2)
= 总分 2
→ uncertain → 跳过
```
