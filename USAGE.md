# 使用指南

## 快速开始

### 1. 首次运行

```bash
# 一键启动（自动安装依赖）
./start.sh
```

### 2. 日常使用

```bash
# 正常模式
python3 full_auto.py

# 调试模式（跳过 AI 判断）
python3 full_auto.py --debug
```

## 配置说明

### config.json

```json
{
  "groupManageId": 6275817,  // 七鱼群组 ID（固定）
  "debug": false              // 是否启用调试模式
}
```

### keywords.yaml

自定义关键词和评分：

```yaml
# 客户确认词
customer_confirm:
  score: 3
  keywords:
    - 好的
    - OK
    - 可以
    - 明白了

# 客户感谢词
customer_thanks:
  score: 2
  keywords:
    - 谢谢
    - 感谢

# 我方承诺词
our_promise:
  score: 2
  keywords:
    - 稍后
    - 明天
    - 下周

# 问题已解决
problem_solved:
  score: 2
  keywords:
    - 已解决
    - 已修复

# 新问题出现（负分）
new_problem:
  score: -5
  keywords:
    - 还有个问题
    - 另外
    - 再问一下

# 评分阈值
thresholds:
  strong_end: 4  // >= 4 分强制关闭
  keep: 0        // < 0 分保留会话
```

### credentials.local.yaml（可选）

如果需要 AI 兜底判断：

```yaml
username: your_email@example.com
password: your_password
```

## 工作流程

### 1. 启动阶段

程序启动后会：
1. 启动 Whistle 代理
2. 打开企微侧边栏
3. 点击"网易智企"标签
4. 提取 token 和 cookie
5. 进入监听循环

**日志示例：**
```
🎯 企微自动结束会话 v1.5 启动
🚀 启动 Whistle...
✅ Whistle 已启动
🧭 启动阶段：强制将 UI 调整到"网易智企就绪态"...
✅ UI 就绪成功
👀 开始监听当前企微会话...
```

### 2. 监听阶段

程序会持续监听：
- 会话切换
- 消息变化

**日志示例：**
```
🔄 检测到会话切换: None -> VIP云信-深圳联友
🔄 刷新侧边栏以加载最新消息...
✅ 已刷新侧边栏
```

### 3. 判断阶段

当检测到消息变化时：
1. 多次采样读取消息（4 次）
2. 选择消息数最多的结果
3. 过滤无效消息
4. 本地关键词评分
5. 如果不确定，调用 AI

**日志示例：**
```
💬 检测到消息变化: VIP云信-深圳联友（消息数: 15）
🔄 开始多次采样读取最新消息...
  采样 1/4: 15 条
  采样 2/4: 15 条
✅ 最终选择: 采样 1 的结果（15 条消息）
📍 当前群/会话: VIP云信-深圳联友
📝 有效消息数: 12
本地判断: strong_end_candidate / 0.80 / 评分5: 客户确认词 + 我方承诺
```

### 4. 关闭阶段

如果判断需要关闭：
1. 重新点击网易智企（刷新数据）
2. 再次提取最新 sessionId
3. 调用七鱼 API 关闭会话

**日志示例：**
```
进入结束会话链路...
🔄 重新切换到网易智企以刷新数据...
✅ 已切换到网易智企
🎯 使用 latest 接口的会话: 15002897771
🔄 关闭会话: VIP云信-深圳联友 / 15002897771
  ✅ 成功
```

## 调试模式

### 启用调试

```bash
python3 full_auto.py --debug
```

### 调试模式特性

1. **跳过 AI 判断**：不调用 Brainmaker
2. **详细日志**：显示更多调试信息
3. **快速测试**：适合测试关键词配置

### 日志示例

```
本地判断: uncertain / 0.50 / 评分0: 无明显信号
🐛 DEBUG 模式：跳过 Brainmaker 调用
最终判断: not_end / 0.50 / DEBUG模式跳过AI
```

## 常见场景

### 场景 1：客户确认问题已解决

**对话：**
```
客户：这个问题怎么解决？
我方：已经修复了，请重试
客户：好的，谢谢
```

**判断：**
- 客户确认词（好的）：+3 分
- 客户感谢词（谢谢）：+2 分
- 总分：5 分 >= 4
- **结果：自动关闭**

### 场景 2：我方承诺稍后答复

**对话：**
```
客户：这个功能什么时候能上线？
我方：我们确认后明天回复你
客户：好的
```

**判断：**
- 我方承诺词（明天回复）：+2 分
- 客户确认词（好的）：+3 分
- 总分：5 分 >= 4
- **结果：自动关闭**

### 场景 3：客户提出新问题

**对话：**
```
客户：这个问题解决了
我方：好的
客户：还有个问题，另外一个功能...
```

**判断：**
- 新问题出现（还有个问题）：-5 分
- 总分：-5 分 < 0
- **结果：保留会话**

### 场景 4：不确定的情况

**对话：**
```
客户：嗯
我方：有其他问题随时联系
```

**判断：**
- 评分：0 分（无明显信号）
- 调用 AI 判断
- **结果：根据 AI 结果决定**

## 故障排查

### 问题 1：未捕获到 qiyukf 请求

**症状：**
```
Whistle qiyukf 请求数: 0
```

**解决：**
```bash
./repair_whistle_env.sh
```
然后重启企业微信。

### 问题 2：消息识别不准

**症状：**
- 提取到的消息是群成员列表
- 或者消息数量不对

**解决：**
1. 检查日志中的路径
2. 确认侧边栏是否正确关闭
3. 尝试手动关闭侧边栏后重试

### 问题 3：关闭会话失败

**症状：**
```
❌ 失败: session is timeout
```

**解决：**
- 程序会自动重试获取最新 sessionId
- 如果持续失败，检查网络连接
- 查看 Whistle 是否正常运行

### 问题 4：AI 调用失败

**症状：**
```
Brainmaker 调用失败: 请创建 credentials.local.yaml
```

**解决：**
1. 创建 `credentials.local.yaml`
2. 填写 Brainmaker 账号密码
3. 或者使用 `--debug` 模式跳过 AI

## 性能优化

### 1. 调整采样次数

编辑 `full_auto.py`：
```python
# 默认 4 次采样
for i in range(4):
```

### 2. 调整主循环延迟

```python
# 默认 1 秒
time.sleep(1)
```

### 3. 调整关键词评分

编辑 `keywords.yaml`，根据实际情况调整评分和阈值。

## 日志说明

### 日志级别

- **INFO**：正常流程
- **WARNING**：警告信息
- **ERROR**：错误信息
- **DEBUG**：调试信息（需要 --debug）

### 日志位置

```
logs/wecom_auto_YYYY-MM-DD.log
```

### 关键日志

- `🎯 企微自动结束会话 v1.5 启动` - 程序启动
- `✅ UI 就绪成功` - 初始化完成
- `💬 检测到消息变化` - 发现新消息
- `本地判断: strong_end_candidate` - 判断需要关闭
- `✅ 成功` - 关闭成功
